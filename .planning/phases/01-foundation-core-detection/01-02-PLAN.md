  <behavior>
HandstandClassifier.classify behavior:
- Input: VNHumanBodyPoseObservation where ankleCenter.y - wristCenter.y >= 0.15 (Vision normalized, Y increases upward) AND abs(ankleCenter.x - wristCenter.x) <= 0.25 AND all 4 joints confidence >= 0.3 → output: .handstand
- Input: Observation where ankles are BELOW wrists (standing pose, ankleCenter.y < wristCenter.y) → output: .notHandstand
- Input: Any joint confidence < 0.3 → output: .insufficientData
- Input: Ankle midpoint horizontally offset from wrist midpoint by > 0.25 → output: .notHandstand
- Thresholds must be configurable via HandstandClassifier.Configuration struct (not hard-coded)

HoldStateMachine.feed behavior:
- .searching + .handstand → .detected(frameCount: 1)
- .searching + .notHandstand → .searching (no change)
- .searching + .insufficientData → .searching (no change)
- .detected(count) + .handstand, count+1 < detectionThreshold → .detected(frameCount: count+1)
- .detected(count) + .handstand, count+1 >= detectionThreshold → .timing(startDate: Date())
- .detected + .notHandstand → .searching (lost during accumulation, reset)
- .detected + .insufficientData → .searching (lost during accumulation, reset)
- .timing + .handstand → .timing (no change, lossFrameCount = 0)
- .timing(startDate) + .notHandstand, lossFrameCount+1 < lossThreshold → .timing (lossFrameCount++)
- .timing(startDate) + .notHandstand, lossFrameCount+1 >= lossThreshold → .ended(duration: elapsed)
- .ended + any → .ended (caller must call reset())
- HoldStateMachine.reset() → .searching, lossFrameCount = 0

Test construction approach for HandstandClassifier:

This makes the geometric logic unit-testable without Vision framework.

**PoseDetector.swift** — actor (not global actor — it processes frames but doesn't own AVCaptureSession):
- `private let bodyPoseRequest = VNDetectHumanBodyPoseRequest()` — SINGLE instance, created once, reused across all frames (per performance mandate — per-frame instantiation forces Vision to reallocate internal model buffers)
- `nonisolated func processFrame(_ sampleBuffer: CMSampleBuffer) -> VNHumanBodyPoseObservation?` — synchronous, called on the video data output queue
- Create `VNImageRequestHandler(cvPixelBuffer: pixelBuffer, options: [:])` per frame (correct — it wraps a specific pixel buffer)
- MUST NOT wrap in Task {} or async/await — Vision.perform() is synchronous by design; async wrapping adds scheduling latency and can cause buffer accumulation
- Pass explicit orientation option: `VNImageRequestHandler(cvPixelBuffer: pixelBuffer, orientation: .up, options: [:])` — .up corresponds to portrait with home button at bottom; required to prevent joint coordinate mismatches
