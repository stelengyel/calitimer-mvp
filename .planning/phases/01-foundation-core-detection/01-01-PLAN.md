**CameraManager.swift** — `@CameraActor final class CameraManager: NSObject` with:
- `private let captureSession = AVCaptureSession()` — keep a property accessor `var session: AVCaptureSession { captureSession }` so CameraPreviewView can bind to it
- `sessionPreset = .medium` — MANDATORY per performance mandates (not .high — Vision doesn't improve above 480p at training distances)
- `private var currentInput: AVCaptureDeviceInput?`
- `private var videoDataOutput: AVCaptureVideoDataOutput?`
- `var onFrame: ((CMSampleBuffer) -> Void)?` — frame callback closure
- `func configure(position: AVCaptureDevice.Position) throws` — uses beginConfiguration/commitConfiguration (NEVER stopRunning/startRunning for switches — causes visible freeze)
- `func start() async` — calls `captureSession.startRunning()` on CameraActor (never on MainActor — startRunning blocks 300-2000ms)
- `func stop() async`
- `func switchCamera() throws` — calls configure() with toggled position; uses beginConfiguration/commitConfiguration NOT stop/start
- Implement `AVCaptureVideoDataOutputSampleBufferDelegate.captureOutput(_:didOutput:from:)` as `nonisolated` — calls `onFrame?(sampleBuffer)`. The buffer MUST NOT be retained after this callback returns (camera pool has 3-6 slots).
- Camera permission check: `func checkPermission() async -> Bool` — handles .authorized, .notDetermined (request), .denied/.restricted (return false)
- Explicit orientation: set `AVCaptureConnection.videoOrientation = .portrait` when adding output connection — MANDATORY to prevent Vision coordinate mismatches

**HapticManager.swift** — Pure utility struct (no state):
```swift
struct HapticManager {
    static func playTargetReached() {
        let generator = UINotificationFeedbackGenerator()
        generator.prepare()
        generator.notificationOccurred(.success)
    }
    static func playHoldDetected() {
        let generator = UIImpactFeedbackGenerator(style: .medium)
        generator.prepare()
        generator.impactOccurred()
    }
}
```

**CameraPreviewView.swift** — UIViewRepresentable wrapping AVCaptureVideoPreviewLayer:
```swift
struct CameraPreviewView: UIViewRepresentable {
    let session: AVCaptureSession

    func makeUIView(context: Context) -> PreviewView {
        let view = PreviewView()
        view.previewLayer.session = session
        view.previewLayer.videoGravity = .resizeAspectFill
        return view
    }

    func updateUIView(_ uiView: PreviewView, context: Context) {}

    class PreviewView: UIView {
        override class var layerClass: AnyClass { AVCaptureVideoPreviewLayer.self }
        var previewLayer: AVCaptureVideoPreviewLayer { layer as! AVCaptureVideoPreviewLayer }
    }
}
```
Give CameraPreviewView a stable identity — the caller must use `.id("camera-preview")` to prevent SwiftUI from destroying and recreating the UIViewRepresentable during re-renders (would tear down the preview layer).
