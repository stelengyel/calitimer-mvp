Create `SkeletonRenderer` as a SwiftUI `View` that wraps a `Canvas`. This view is the ONLY view that reads `coordinator.recognizedPoints` — keeping it isolated prevents the full SessionView hierarchy from re-rendering at the skeleton update rate (10Hz).

**SkeletonRenderer** accepts:
- `let coordinator: SessionCoordinator` — reads `recognizedPoints` and `isSkeletonVisible`
- Do NOT read any other coordinator property in this view's body

**Coordinate transform:** Vision normalized coordinates have origin at bottom-left (0,0) with Y increasing upward. SwiftUI has origin at top-left with Y increasing downward. Transform required:
```swift
func transformPoint(_ vnPoint: CGPoint, in size: CGSize) -> CGPoint {
    // Flip Y, scale to view size
    // NOTE: for rear camera, no mirroring needed
    // For front camera (mirrored preview), flip X: x = 1.0 - vnPoint.x
    // Determine camera position from coordinator.cameraManager.currentPosition
    let flippedY = 1.0 - vnPoint.y
    let x = isFrontCamera ? (1.0 - vnPoint.x) : vnPoint.x
    return CGPoint(x: x * size.width, y: flippedY * size.height)
}
```

**Bone connections** (from RESEARCH.md code example — use exactly these pairs):

**Canvas rendering:**
```swift
Canvas { context, size in
    for (from, to) in bones {
        guard let p1 = transformedPoints[from], let p2 = transformedPoints[to] else { continue }
        var path = Path()
        path.move(to: p1)
        path.addLine(to: p2)
        context.stroke(path, with: .color(.green.opacity(0.9)), lineWidth: 3)
    }
    for (_, point) in transformedPoints {
        let rect = CGRect(x: point.x - 5, y: point.y - 5, width: 10, height: 10)
        context.fill(Circle().path(in: rect), with: .color(.yellow))
    }
}
```

Bone color: `.green` (per Claude's discretion — high contrast on camera feed). Joint color: `.yellow`. Line width: 3pt.

`isVisible` guard: wrap entire Canvas in `if coordinator.isSkeletonVisible { ... }`. Do not render at all when hidden — no empty canvas.

Pre-compute `transformedPoints` dict from `coordinator.recognizedPoints` before passing to Canvas. Only include points with confidence >= 0.3 (filter low-confidence joints).

The view must be `.allowsHitTesting(false)` — it's a transparent overlay, not interactive.

**DetectionStateOverlay.swift** — colored ring driven by HoldStateMachine.State:

The ring is the primary at-a-distance feedback. Make it unmissable: thick border (6pt), high saturation colors.

```swift
struct DetectionStateOverlay: View {
    let coordinator: SessionCoordinator  // reads holdState, isDetectionStateVisible, currentHoldDuration

    @State private var showingHoldResult: Bool = false
    @State private var lastHoldDuration: TimeInterval = 0

    var body: some View {
        if coordinator.isDetectionStateVisible {
            ZStack {
                // Detection ring
                RoundedRectangle(cornerRadius: 0)
                    .strokeBorder(ringColor, lineWidth: 6)
                    .ignoresSafeArea()
                    .animation(.easeInOut(duration: 0.2), value: ringColor)

                // Hold result flash on .ended state
                if showingHoldResult {
                    VStack {
                        Spacer()
                        Text(String(format: "%.1fs", lastHoldDuration))
                            .font(.system(size: 48, weight: .bold, design: .monospaced))
                            .foregroundStyle(.white)
                            .shadow(radius: 4)
                            .padding(.bottom, 120)
                    }
                    .transition(.opacity)
                }
            }
            .onChange(of: coordinator.holdState) { oldState, newState in
                if case .ended(let duration) = newState {
                    lastHoldDuration = duration
                    withAnimation { showingHoldResult = true }
                    Task {
                        try? await Task.sleep(for: .seconds(2))
                        withAnimation { showingHoldResult = false }
                    }
                }
            }
        }
    }

    private var ringColor: Color {
        switch coordinator.holdState {
        case .searching:           return Color.gray.opacity(0.6)
        case .detected:            return Color.yellow
        case .timing:              return Color.green
        case .ended:               return Color.gray.opacity(0.6)
        }
    }
}
```

Per CONTEXT.md locked decisions:
- searching → gray ring
- detected (accumulating) → yellow ring
- timing (active hold) → green ring
- ended: ring briefly flashes (show hold duration for 2s) then resets to gray
- When detection indicator toggled OFF: ring completely disappears
