**Internal:**
```swift
private var stateMachine = HoldStateMachine()
private let classifier = HandstandClassifier()
private let poseDetector = PoseDetector()

private var frameCounter = 0            // For throttling skeleton updates
private var targetAlertFired = false    // Prevent repeated alerts

```

**`func startSession(targetDuration: TimeInterval) async`:**
1. Set `isSessionActive = true`, `completedHolds = []`, `targetAlertFired = false`
2. Reset `stateMachine`, `currentHoldDuration = 0`, `holdState = .searching`
3. Ensure camera configured (await `cameraManager.configure(position: .back)` if needed)
4. Assign `cameraManager.onFrame = { [weak self] sampleBuffer in self?.processFrame(sampleBuffer) }` — NOTE: this closure is called on videoDataOutputQueue, not MainActor
5. `await cameraManager.start()`

**`func endSession() async`:**
1. Set `isSessionActive = false`
2. Cancel `timerTask`
3. `await cameraManager.stop()`
4. `cameraManager.onFrame = nil`

**`nonisolated func processFrame(_ sampleBuffer: CMSampleBuffer)`** — called on videoDataOutputQueue:
- Call `poseDetector.processFrame(sampleBuffer)` synchronously (nonisolated, synchronous by design)
- Classify observation with classifier
- `frameCounter += 1`
- Throttle: only update skeleton on MainActor every 3rd frame (10Hz effective at 30fps). Throttle `recognizedPoints` delivery to MainActor — MANDATORY per performance mandates.
- For holdState transitions: only dispatch to MainActor when state ACTUALLY CHANGES (not every frame) — check `classification != previousClassification` or use a local copy
- Batch skeleton points and hold state changes into a SINGLE MainActor hop when they coincide
- Pattern: `Task { @MainActor in self.updateState(newState, points: points) }`

**`@MainActor private func updateState(_ newState: HoldStateMachine.State, points: [VNHumanBodyPoseObservation.JointName: VNRecognizedPoint]?)`:**
- Feed newState into stateMachine (re-feed on MainActor using the classified result, not by calling processFrame again — pass classification result)
- Actually: since stateMachine.feed() needs to happen and its result consumed atomically on MainActor, the approach is:
  1. On videoDataOutputQueue: classify only (get classification result)
  2. MainActor hop: `stateMachine.feed(classification)`, then react to new state
- When state enters `.timing`: start timerTask (ContinuousClock-based, 10Hz), call `HapticManager.playHoldDetected()`, call `UIFeedbackGenerator.prepare()` was already called when entering `.detected`
- When state enters `.detected`: call `UIImpactFeedbackGenerator(style:.medium).prepare()` — MANDATORY: prepare() must be called at least one runloop cycle before firing haptic
- When state enters `.ended(let duration)`: cancel timerTask, record hold in completedHolds, reset stateMachine, reset currentHoldDuration = 0

**Timer task (10Hz, ContinuousClock-based):**
```swift
timerTask = Task { @MainActor [weak self] in
    let clock = ContinuousClock()
    let start = clock.now
    while !Task.isCancelled {
        try? await Task.sleep(for: .milliseconds(100))
        guard let self else { return }
        let elapsed = clock.now - start
        self.currentHoldDuration = elapsed.timeInterval
        // Check target alert
        if self.currentHoldDuration >= self.targetDuration && !self.targetAlertFired {
            self.targetAlertFired = true
            HapticManager.playTargetReached()
            // Play beep sound (see below)
        }
    }
}
```
MUST use ContinuousClock, NOT Date() (monotonic, immune to system clock adjustments).
MUST NOT tie timer updates to frame delivery — timer task runs independently.

**Target beep sound:** Use `AudioServicesPlaySystemSound(1057)` (system "tink" beep — classic timer sound) from AudioToolbox framework. Import AudioToolbox. This is a system sound, no asset required. The user asked for "classic timer beep" — system sound 1057 is the SMS received sound (short, audible, gym-appropriate). If preferred: sound 1005 (key press) or 1016 (tweet). Use 1057 as default. (Per Claude's discretion — exact sound asset.)

**Important wiring constraint:** `SessionView.body` MUST NOT read `recognizedPoints` — only the future `SkeletonOverlayView` (Plan 04) reads it. Pass coordinator to child views directly; each child reads only what it needs.
