**Changes to CameraManager.swift:**

1. Add a private video output queue (separate from the existing camera queue to avoid blocking):
```swift
private static let videoOutputQueue = DispatchQueue(label: "com.calitimer.videoOutput", qos: .userInitiated)
```

2. Add stored properties:
```swift
nonisolated(unsafe) private var videoOutput: AVCaptureVideoDataOutput?
// VisionProcessor lives on @CameraActor — bridge via Task
private var visionProcessor = VisionProcessor()
```

`visionProcessor` is the published source for LiveSessionView. Make it accessible: `let visionProcessor: VisionProcessor` (or `@Published` if needed — prefer `let` since it's created once).

3. In `configureAndStart()`, after adding input and before `startRunning()`, add output:
```swift
let output = AVCaptureVideoDataOutput()
output.videoSettings = [kCVPixelBufferPixelFormatTypeKey as String: kCVPixelFormatType_420YpCbCr8BiPlanarFullRange]
output.alwaysDiscardsLateVideoFrames = true
// Delegate set nonisolated — CameraManager is @unchecked Sendable, safe
output.setSampleBufferDelegate(self, queue: Self.videoOutputQueue)
if session.canAddOutput(output) {
    session.addOutput(output)
    self.videoOutput = output
}
```

4. Make `CameraManager` conform to `AVCaptureVideoDataOutputSampleBufferDelegate`:
```swift
extension CameraManager: AVCaptureVideoDataOutputSampleBufferDelegate {
    nonisolated func captureOutput(_ output: AVCaptureOutput,
                                   didOutput sampleBuffer: CMSampleBuffer,
                                   from connection: AVCaptureConnection) {
        let processor = visionProcessor
        Task { @CameraActor in
            processor.process(sampleBuffer: sampleBuffer)
        }
    }
}
```

`nonisolated` is required because the delegate callback arrives on `videoOutputQueue` (not MainActor). Accessing `visionProcessor` as `let` from `nonisolated` context is safe because `let` properties are immutable.

Do NOT use `@CameraActor` on the delegate method — the delegate callback is dispatched by AVFoundation on the provided queue, not by the actor. Bridging via `Task { @CameraActor in }` is the correct pattern.
